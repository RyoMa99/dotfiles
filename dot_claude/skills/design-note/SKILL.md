# design-note

会話で議論した設計方針・意思決定を ideas リポジトリの `design/` ディレクトリに記録する。

## Step 1: 前提チェック

まず Bash で `ghq root` を実行してリポジトリルートを取得する。以降のステップではその出力を `{GHQ_ROOT}` として使用する。

次に ideas リポジトリの存在を確認する。

```bash
[ -d {GHQ_ROOT}/github.com/RyoMa99/ideas ] && echo "OK" || echo "NG: ideas リポジトリが見つかりません"
```

ideas リポジトリが存在しない場合は動作しない。ユーザーに `ghq get github.com/RyoMa99/ideas` を案内して終了する。

## Step 2: ファイル名の決定

会話のトピックを日本語で簡潔に表すファイル名を決める。

- 形式: `{トピックの説明}.md`（スペースなし）
- 例: `優先度のある2値設計パターン.md`、`マルチテナント認証設計.md`
- 既存ファイルと重複する場合はユーザーに確認する

## Step 3: まとめの作成

会話から Markdown を作成する。**簡潔さより再現性を優先する**（後から読んだ人が議論の経緯と結論を理解できること）。

### 構成テンプレート

```markdown
# {トピック}

> {1〜2行の文脈説明。なぜこの設計判断が必要になったか}

---

## {セクション}

（比較表・コード例・フロー図を積極的に使う）

## まとめ / 判断フロー

決定の要点を箇条書きやフローで整理する。
```

セクション構成は内容に応じて柔軟に変える。今回の会話で登場したコード例は積極的に再掲する。

## Step 4: 保存

`{GHQ_ROOT}/github.com/RyoMa99/ideas/design/` に保存する。ディレクトリがなければ `mkdir -p` で作成する。

Write ツールでファイルを書き出し、完了後に保存したフルパスをユーザーに伝える。
