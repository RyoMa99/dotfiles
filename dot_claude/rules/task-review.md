# タスクレビューのルール

## タスク分割時
- 各タスクにレビューポイント（🔍）を設ける
- リファクタ完了、store作成、UI実装などの区切りごとにチェックポイントを置く
- **実装タスクの前にガードレール準備タスクを配置する**

## ガードレール先行（実装開始前に確認）
- [ ] テスト環境・テストファイルの準備
- [ ] 型定義・スキーマ定義

> AIが高速にコードを生成する前に、品質を担保する仕組みを先に用意する

## タスク実行時
- 各レビューポイントで `AskUserQuestion` を使いユーザーの確認を取る
- **ユーザーのOKなしに次タスクへ進まない**
- 確認時には変更差分・lint結果・Storybook確認依頼など具体的な情報を提示する
